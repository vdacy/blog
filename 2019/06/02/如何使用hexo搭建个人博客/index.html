如何使用Hexo搭建个人博客

Posted on 2019-06-02

简介

现如今，越来越多的技术人员都搭建了自己的技术博客。此次就借用github的服务器，使用 Hexo 做为博客搭建的框架，将博客托管到 Github Pages 上，他支持配置自定义域名，可以在该页面中绑定，当放域名时，会自动映射到 Github 的个人博客上。

安装 Hexo

安装hexo必须需要在电脑上先安装 Node.js、Git，后才能安装 Hexo

安装Node.js

我使用的是 win10 系统进行安装的，安装 Node.js 有个诀窍,那就是 闭上眼 ，一直点击 一步(next)就可以了，当然安装路径根据自己需要进行修改。

下载链接：Node: https://nodejs.org/en/download/

安装Git

此处也是 win10 进行安装的，由于已经安装过了，需要安装的朋友可以参考百度。

下载链接：Git：https://git-scm.com/downloads

Git与Github进行绑定

没有账号github账号的先注册一下: https://github.com/

安装成功后，使用鼠标右键选择 Git Bash Here，执行下面三个命令

使用命令生成ssh密钥，默认不需要设置密码，看到信息后三个回车，生成后会在本机用户下生成密钥文件（id_rsa、id_rsa.pub），我的生成路径为：C:\Users\zu101.ssh

Github创建仓库绑定密钥

这里使用测试仓库进行演示，新建一个用于存放个人博客的仓库，后再进入用户设置


设置ssh

必备的 NodeJs、Git 安装后，接下来使用 npm 命令进行安装，安装有一定过程，请等待安装完成

hexo init blog
cd blog
npm install

现在我们本地就可以访问了，但是要真正在网络上进行访问仍需要一些配置，首先我们先配置下 hexo 的配置文件，我们进入刚初始化的文件夹，进入 _config.yml进行配置

仅仅配置这个是不够的，我们需要在仓库中设置 GitHub Pages，配置成功后才能在网络上进行访问

  1. 首先我们先进入创建的仓库中
  2. 进入仓库设置 Settings
  3. 设置GitHub Pages

进入仓库的设置中，页面往下拉，找到 GitHub Pages，选择master branch

将Settings中的Repository name 改成 你的giuhub账户名/github.io

配置成功后你将会看到 Your site is published at https://name.github.io,就可以在网络中进行访问了

如果需要配置自定以域名，可以在github上进行如下配置，没有域名的可以到aliyun 上购买，进行实名认证后通过域名解析，后将repository name 填入，还可以在阿里云上申请免费的SSL证书哦
